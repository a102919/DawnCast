# DawnCast 資訊架構（IA）

> 狀態：pre-product 提案。以下架構從 5 位 B1 用戶的「心智模型」反推，不是從內部功能模組切。
> 紅線提醒：這是構想藍圖，不是評估既有介面。

---

## 一、從訪談反推的心智模型（設計前提）

在分群之前，先把訪談裡反覆出現的「用戶怎麼想」攤開，因為 IA 的對錯由這些心智決定：

1. **「我要的是廣播，不是課程」**（宜君、佳穎）
   用戶把這件事歸類在「背景音 / 通勤陪伴」，不是「打開一個學習 App 上課」。任何長得像課程儀表板、有「進度條逼你打卡」的東西，都會被歸到「殭屍 App」那一類心智抽屜。
   → IA 不能以「課程 / 單元 / 章節」當頂層。

2. **「斷掉不能有罪惡感」**（宜君、佳穎）
   連續打卡天數、紅色未讀數字、「你已經 N 天沒來」這類 streak 機制，是明確的流失觸發器，不是黏著工具。
   → IA 裡不存在「打卡」「連續天數」這種頂層概念。回來時看到的是「今天這集」，不是「你斷了 14 天」。

3. **「主題要對到我，否則不換」**（冠廷、Kevin、佳穎）
   付費意願的開關不是「英語學習」，而是「這個主題是我本來就想聽的（科技 / 商業 / 我的工作場景）」。Kevin 的原話：「你跟 All Ears English 差在哪裡，先回答我。」
   → 主題 / 場景必須是頂層的、可見的入口，不能藏在分類三層下面。

4. **「沒有字幕我就跟不上，但只看字幕又不算練聽力」**（詩涵、佳穎）
   B1 的甜蜜點是：能聽、聽不懂時能即時對照、查完能回到聽。字幕 / 逐字稿 / 查詞不是進階功能，是地板需求。
   → 「跟讀 / 對照」要在「聽」這件事的同一層，不是設定頁裡的開關。

5. **「我需要某種確認自己有在進步的感覺」**（全員）
   五個人都講了「不知道有沒有進步」。但他們要的不是分數報表（那像考試、有壓力），而是輕量的「你有在動」訊號——學過哪些詞、聽過幾集、這週比上週多接觸了什麼。
   → 「進步感」要存在，但要做成「回顧 / 收穫」的語氣，不是「成績單」。

6. **「難度要剛好，但我沒耐心自己篩」**（全員）
   太難放棄、太簡單幼稚，中間甜蜜點難找，而且「沒耐心一直試聽篩選」。
   → 難度是系統幫你定錨的，不是一個要用戶自己調的設定旋鈕；頂多給「再簡單一點 / 再難一點」的微調。

## 二、頂層導覽（topLevelNav）

只設 4 個頂層，刻意少。理由：用戶把這歸類為「廣播級的零阻力事物」，導覽越多越像「要我認真上課的 App」，反而觸發抗拒。

| 頂層 | 用戶心裡的話 | 對應 |
|------|-------------|------|
| **今天** | 「打開就能聽，不用想要聽什麼」 | 單集播放 + 今日推薦，預設落地頁 |
| **探索** | 「我想挑我本來就想聽的主題」 | 依主題 / 場景瀏覽（科技、商業、文化、科學、職場場景） |
| **我的收穫** | 「我到底有沒有在進步、學過哪些詞」 | 查過的詞、聽過的集、輕量回顧（非成績單、非 streak） |
| **設定** | 「難度、語速、訂閱，這些我偶爾才動」 | 難度錨點、語速、訂閱管理、通知（預設關 streak 型通知） |

> 註：刻意**不設**「進度 / 打卡 / 排行榜 / 課程」頂層——這些是訪談裡的流失觸發器。

## 三、卡片分類理由（cardSortingRationale）

模擬一次開放式卡片分類：把產品所有候選功能（單集播放、雙語字幕、點擊查詞、生字本、主題瀏覽、職場場景包、難度調整、語速調整、聽過記錄、本週回顧、訂閱方案、下載離線、被打斷後續播）寫成卡片，依「用戶會把它跟誰放一起」分群，而非依「工程上它屬於哪個模組」。

- **為什麼「雙語字幕 / 點擊查詞 / 生字本」不獨立成頂層，而黏在「播放」與「我的收穫」兩端？**
  訪談顯示「查詞」不是一個獨立目的地，它是「聽到不懂 → 當下對照 → 之後想複習」這條線上的兩個點。查的動作屬於「聽」的當下（黏在播放器），複習屬於「回顧」（黏在我的收穫）。把它做成獨立頂層會切斷這條心智線。

- **為什麼「難度 / 語速」放進設定，而不是放在播放器顯眼處？**
  甜蜜點難找，但用戶「沒耐心一直篩」。如果把難度當主要操作擺在播放器，等於把篩選成本丟回給用戶——這正是他們放棄 YouTube 的原因。所以難度由系統定錨，設定裡只留「整體再簡單 / 再難一點」的低頻微調。

- **為什麼「主題 / 場景」要從功能升格成頂層（探索）？**
  因為它是付費開關。對冠廷、Kevin、佳穎，「換不換」取決於「主題是不是我本來就想聽的」。把主題藏在分類深處等於把產品最強的差異點藏起來。職場場景（委婉拒絕、會議插話、技術對接、客戶 small talk）要當「探索」裡的一級公民，因為那是「學了會用到」的具體錨點。

- **為什麼「本週回顧」歸到「我的收穫」而不是做成 streak / 儀表板？**
  五人都要「進步感」，但宜君、佳穎明確排斥「連續天數」「逼你打卡」。所以進步訊號要存在、但語氣是「你這週順手學了這些」，不是「你達成 / 你中斷了」。它跟「查過的詞、聽過的集」是同一個心智抽屜：回頭看看自己累積了什麼。

- **為什麼「被打斷後續播 / 下載離線」不單獨成群，而隱形地內建在播放？**
  冠廷提到「電話進來 context 就斷了，撿不回來」。這是體驗預設行為（自動記位置、回來給一句『上次聽到這裡』），不該是一個要用戶管理的功能頁。好品味：把邊界條件做成正常流程，而不是給它一個設定頁。

## 四、分群結果（contentGroups）

### 今天（預設落地）
- 今日單集（一鍵播放，預設就緒）
- 上次聽到這裡（被打斷後的續播入口）
- 一句話「這集帶你熟什麼」（聽前的一行收穫預告，回答「我花這 3 分鐘換到什麼」）

### 探索（主題 / 場景）
- 依主題瀏覽：科技、商業、文化、科學
- 依職場場景瀏覽：會議插話、委婉拒絕、技術對接、客戶 small talk、簡報開場
- 已上架集數列表（可挑、可預覽一行摘要）

### 聽當下（黏在播放器，不是獨立頂層）
- 雙語字幕 / 逐字稿對照
- 點擊查詞（聽到不懂 → 點 → 釋義，不離開播放）
- 加入生字本（一鍵收藏該詞 / 該句型）
- 語速微調（當下覺得太快可降速，不必進設定）

### 我的收穫（輕量、非成績單、非 streak）
- 我的生字本（查過 / 收藏的詞與句型）
- 聽過的集（自然累積的紀錄，無「連續天數」框架）
- 本週收穫回顧（「你這週順手熟了這些」語氣，可關閉）

### 設定（低頻）
- 難度錨點（整體再簡單 / 再難一點）
- 預設語速
- 訂閱方案（Free / Pro 管理、取消入口清楚——回應「怕忘記取消」焦慮）
- 通知（預設只開「新集上架」，不開 streak 型催促）
- 離線下載（給通勤 / 開車的弱網場景）
