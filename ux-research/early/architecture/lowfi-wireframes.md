# DawnCast 低保真線框「文字規格」

> 只談**版面區塊**與**資訊層級**，不談顏色 / 字體 / 視覺風格。pre-product 構想。
> 每個畫面標註它在回應訪談裡的哪個心智，避免設計漂移。

---

## 畫面 1：今天（預設落地頁）

**設計意圖**：打開即可聽，零思考成本。對抗「頭很大就放棄」「全英文聽 30 秒就換掉」。

**layoutSpec（由上到下）**
1. 頂部：極簡。只有「今天」字樣與日期，無 streak、無連續天數、無紅點。
2. 主視覺區（畫面中央、最大層級）：今日單集卡。包含
   - 集名 + 主題標籤（如「職場場景 · 會議插話」）
   - 一行「這集帶你熟什麼」（聽前收穫預告，最高文字權重的說明文字）
   - 約 3 分鐘的時長標示
3. 主行動區（緊接主視覺、視覺權重次高）：大尺寸播放鍵。
4. 續聽帶（條件式，只有上次被打斷才出現）：「上次聽到這裡 · 繼續」。
5. 底部：4 格頂層導覽（今天 / 探索 / 我的收穫 / 設定），今天為選中態。

**primaryAction**：播放今日單集（一鍵，無前置設定）

## 畫面 2：播放中（雙語字幕 + 點擊查詞）

**設計意圖**：聽得下去、聽不懂時能即時對照、查完回到聽。B1 核心價值畫面。對抗「沒字幕跟不上 / 只看字幕不算練聽」「查完沒真的學」。

**layoutSpec（由上到下）**
1. 頂部：集名 + 一行收穫預告（縮小版，提醒「我在聽什麼」）。
2. 字幕主區（畫面主體、最大層級）：雙語逐字稿，當前句高亮並自動捲動。
   - 英文句為主層級，中譯為次層級（視覺從屬，支援「先看英文、需要才看中文」）。
   - 句中可點的詞有可點性提示（非顏色描述，指「可互動」的層級線索）。
3. 查詞浮層（點詞時由下浮出，覆蓋底部、不蓋住當前字幕、播放不停）：
   - 詞 + 繁中釋義 + 本集語境例句
   - 「加入生字本」次行動
4. 播放控制列（固定底部上方）：播放 / 暫停、前後跳句、語速微調（當下可降速，不必進設定）。
5. 模式切換（小入口）：「盲聽 / 顯示字幕」切換，回應「看字幕就不算練聽力」的兩難。

**primaryAction**：點擊不懂的詞 → 即時釋義（播放不中斷）

## 畫面 3：探索（主題 / 職場場景挑集）

**設計意圖**：讓有 podcast 習慣的人找到「對到我工作場景」的集，觸發付費開關。對抗「主題脫節」「切換成本高」。

**layoutSpec（由上到下）**
1. 頂部：「探索」標題。
2. 場景入口區（先呈現、權重高於學科）：職場場景橫向群組——會議插話 / 委婉拒絕 / 技術對接 / 客戶 small talk / 簡報開場。每格一個場景標籤。
3. 學科主題區（其次）：科技 / 商業 / 文化 / 科學。
4. 集數列表（選定場景 / 主題後展開）：每集一列，含
   - 集名 + 一行「這集帶你熟什麼」
   - 時長 + Free / Pro 標示（Pro 集明示但不擋瀏覽）
5. 無篩選器旋鈕：難度由系統錨定，不丟篩選成本給用戶。

**primaryAction**：選定一個職場場景 → 進入該場景集數列表

## 畫面 4：我的收穫（輕量進步感，非成績單 / 非 streak）

**設計意圖**：給「我有在進步」的確認，但不製造考試壓力與斷裂愧疚。對抗「不知道有沒有進步」「斷掉重啟罪惡感」。

**layoutSpec（由上到下）**
1. 頂部：「我的收穫」標題。**不含**連續天數、不含「你已 N 天沒來」。
2. 本週回顧帶（最上、語氣中性正向）：「你這週順手熟了 N 個詞、聽了 M 集」。可關閉，無斷裂指責。
3. 生字本區（主體）：查過 / 收藏的詞列表，每個詞帶脈絡——「出自哪一集、哪句話」，可點回到原句。
4. 聽過的集區：自然累積的清單，無進度條框架、無「未完成」紅字。
5. 底部導覽：我的收穫為選中態。

**primaryAction**：點生字本中的詞 → 回到當初那一集的那句話（帶脈絡複習）

## 跨畫面共通原則（線框層級守則）

- **不存在的元件（刻意排除）**：連續打卡天數、未讀紅點催促、課程進度條、排行榜、考試型分數報表。它們在訪談裡全是流失觸發器。
- **永遠存在的元件**：一行「這集帶你熟什麼」（回答「我花這時間換到什麼」）、底部 4 格導覽、播放當下的查詞入口。
- **訂閱 / 取消**：訂閱頁把「如何取消」寫在顯眼層級，不藏——把可取消性當信任資產。
