# DawnCast 中期 UX 研究 — Affinity Map

> 對象：5 位付費 Pro 用戶（訂閱滿 3 個月）
> 方法：跨用戶 Affinity Mapping（親和圖）+ 量化問卷交叉驗證
> 受訪者：p1 林佳穎（行銷／通勤捷運）、p2 陳冠廷（竹科工程師／開車）、p3 黃詩涵（大四生／備考多益）、p4 蘇怡君（全職媽媽／做家事多工）、p5 Kevin 張凱文（業務主管／開車通勤）

---

## 1. 痛點叢集（Pain Point Clusters）

跨 5 位用戶歸納，依嚴重度與出現頻率排序。分母統一為 5 人。

### 1.1 學習進度完全不可見（無統計／無 streak）— severity 4
- **頻率**：5/5（全員）
- **uiLocation**：全 app 缺少學習統計／進度儀表板頁面；首頁、單字本頁、設定頁皆無數據
- **量化佐證**：progressFeedbackSatisfaction 平均 ≈ 3（p1=2、p3=4、p4=4、p5=3、p2=7）
- **代表性引言**：
  - p1：「app 完全沒有告訴我『你進步了』，沒有任何進度追蹤……我只能自己感覺，而自己感覺是很主觀的，容易放棄的那種。」
  - p5：「我連自己『有沒有在進步』都說不清楚，要怎麼說服自己繼續訂？」
- **影響**：直接連動到流失、口碑無法傳播（無法向他人背書）。這是全研究最高優先的結構性缺口。

### 1.2 AB 段落重複功能找不到入口（隱藏功能）— severity 3
- **頻率**：5/5（全員提到，付了錢卻用不到）
- **uiLocation**：播放器頁進度條／控制欄，AB 段落重複入口不明顯，無視覺提示
- **代表性引言**：
  - p5：「AB 段落重複完全找不到入口，我還以為這功能不存在……設計上等於隱藏功能。」
  - p4：「不知道要長按還是點兩下，試了半天放棄。」
- **影響**：付費功能曝光度為零＝感知價值流失，也讓用戶質疑「我付的錢值不值」。

### 1.3 閃卡入口太深 + 與播放器脫節 — severity 3
- **頻率**：5/5（全員 SRS 都停用，導航深度是共同原因之一）
- **uiLocation**：閃卡入口藏在「單字本頁 → 開始閃卡複習」二級頁面；底部導航 4 tab 無閃卡入口
- **代表性引言**：
  - p1：「閃卡頁面要進去太多步驟……閃卡介面跟播放器是兩個世界，完全脫節。」
  - p2：「停車後要進去閃卡還要找入口，每次都想說等等再說，結果就沒再用了。」
- **影響**：每日可回訪的核心行動（複習）被導航埋沒，無法承擔習慣入口角色。

### 1.4 單字本平鋪無分類／無篩選 — severity 3
- **頻率**：3/5（p1、p3、p5；p4 vocabUsability=6 較中性）
- **uiLocation**：單字本頁列表，無集數／日期／主題分組，搜尋需打字
- **量化佐證**：vocabBookUsability 平均 ≈ 4.6（p1=4、p3=4、p5=5）
- **代表性引言**：
  - p3：「80 幾個單字全部平鋪，沒有分類、沒有按集數分，捲起來很痛苦，看了就不想整理。」
  - p1：「單字多了之後滑起來很累，找不到想找的那個。」

### 1.5 SRS 排程無彈性，積壓變成「欠債壓力」— severity 3
- **頻率**：3/5（p5 最明顯、p4、p1 脈絡相關）
- **uiLocation**：閃卡複習頁的到期卡片提示與出牌邏輯，無「休息日／延後」機制
- **代表性引言**：
  - p5：「回來看到 40 幾張到期卡，壓力太大直接關掉，之後再也沒打開……是『欠了一堆沒還』的感覺讓我逃避。」
  - p4：「已記住的單字仍一直出現，感覺像考試……那之後就再也沒開過閃卡。」
- **影響**：留存機制（SRS）反而變成驅趕用戶的負向設計。

### 1.6 內容情境無主題分類／篩選 — severity 2
- **頻率**：4/5（p1 職場、p2 主題多樣、p3 面試、p5 商業）
- **uiLocation**：首頁集數卡列表 / 完整曲庫頁，僅時間倒序，無主題標籤篩選
- **代表性引言**：
  - p1：「我現在不知道哪些集是跟『開會』或『寫 email』有關的……不要讓我自己從標題猜。」
  - p5：「現在只能一個一個滑時間軸猜主題，效率超差。」

### 1.7 純聽覺／開車情境無對應設計 — severity 3（對開車族）
- **頻率**：2/5（p2、p5 開車族；但對其族群是高嚴重度）
- **uiLocation**：播放器頁整體互動假設「盯著螢幕、手持操作」，無車用模式
- **代表性引言**：
  - p2：「我大部分時間是把手機放在車架上開車，能用的只有耳朵。你設計的那些查詞、閃卡、AB 重複，都是需要動手的。」（AB 重複[已取消]）
- **競品交叉**：通勤注意力競品複合體指出 60%+ 時間手機在鎖屏狀態，查詞 popup 假設不成立。

### 1.8 播放速度設定不跨集記憶（疑似 bug）— severity 2
- **頻率**：2/5（p3、p5 明確點名）
- **uiLocation**：設定頁有預設速度但播放器實際每次回到 1.0x，設定無效
- **代表性引言**：
  - p5：「設定頁設了預設值但實際開播放器每次還是預設 1.0x，我懷疑那個設定根本沒效果。」

### 1.9 查詞 popup 關閉後進度條跳動（疑似 bug）+ 誤觸 — severity 2
- **頻率**：2/5（p2 進度條跳動、p1 通勤晃動誤觸）
- **uiLocation**：播放器頁字幕區查詞 popup 的關閉行為 / 進度條
- **代表性引言**：
  - p2：「查詞 popup 關掉後，進度條有時候會往前跳一點點……每次都要重新找回剛剛聽到的位置。」

### 1.10 底部導航無播放器快速回入口 — severity 2
- **頻率**：2/5（p2、p5）
- **uiLocation**：底部導航（首頁／單字本／方案／設定），4 tab 皆非播放器
- **代表性引言**：
  - p2：「從單字本切回播放器要先回首頁找那集再點進去……每次都要繞一圈。」

## 2. 黏著動因（Stickiness Drivers）

什麼讓付費用戶 3 個月後還在用？

### 2.1 查詞 popup（最難割捨的核心功能）
- **頻率**：5/5（全員 Q11「最不可取代的功能」答案一致都是它）
- **量化佐證**：wordPopupUsefulness 平均 ≈ 9（p1=9、p2=9、p3=7、p4=9、p5=9）
- **代表性引言**：
  - p3：「這個如果沒有了，我聽 podcast 查字就要一直切 app，超煩。」
  - p5：「點一下秒出、有 IPA 有詞性有中文、還能直接加進單字本……這個是我在別的 app 上體驗不到的。」
- **designImplication**：唯一真正的護城河錨點，任何改版禁止增加其摩擦。可借鑑競品：加入單字本後 popup 0.8 秒自動消失、不中斷播放；點擊字幕單字時自動暫停並跳回該句起點，把查詞變成一次「句子 re-listen」。

### 2.2 完整曲庫 + 無限查詞的組合（付費主因）
- **頻率**：5/5（全員 proFeaturesActuallyUsed 都列入這兩項）
- **代表性引言**：
  - p3：「完整曲庫和無限查詞讓我願意付費……我能隨時補齊之前漏掉的內容。」
- **designImplication**：付費價值由「解鎖 + 即查即用」共同支撐，付費牆是轉換觸發點（多人因 Free 查詞次數用完／鎖頭被擋而付費）。維持但避免錨定效應反噬（VoiceTube 教訓）。

### 2.3 習慣觸發點：通勤／家務的固定情境
- **頻率**：5/5（p1 捷運 25 分、p2/p5 開車、p3 公車、p4 做家事）
- **代表性引言**：
  - p1：「我沒時間學英文，但我有 25 分鐘的通勤。DawnCast 解決的就是這個。」
- **designImplication**：習慣完全外掛在外部情境上，app 本身無觸發機制；情境一斷即流失。需自建回訪鉤子（推播、streak、斷點續播）。

### 2.4 回報感來源：「聽完馬上能用」的閉環體驗
- **頻率**：3/5（p1 拒絕邀請 email、p2 採購談判、p5 anchor 談判詞）
- **代表性引言**：
  - p1：「有一集在講 email 拒絕邀請怎麼說，我當天下午就用到了，真的記住了。」
- **designImplication**：情境化內容 + 即查即用產生真實記憶與價值感，是最強正向回報。應放大職場／商務情境內容並做主題標籤，讓用戶主動找到可即用集數。

### 2.5 字幕中英對照 + 當前句高亮
- **頻率**：4/5（p1、p2、p3、p4 均提到確認聽懂的滿足感）
- **量化佐證**：subtitleReadability 平均 ≈ 8.2
- **代表性引言**：
  - p2：「字幕英文大字高亮當前句，停紅燈的時候瞄一眼很有用，設計對了。」
- **designImplication**：高亮是低成本高滿足設計，可進一步支援「點擊當前句字幕＝從該句重播」的精聽互動（競品共識需求）。

### 2.6 離線下載（開車／隧道族剛需） [已取消]
- **頻率**：2/5（p2、p5；p2 視為最不可取代功能之一）
- **代表性引言**：
  - p2：「隧道裡完全不中斷……如果沒有這個，我大概就要回去聽中文 Podcast 了。」
- **designImplication**：對特定通勤族是留存錨點，下載管理入口應更易發現（p5 抱怨找不到下載管理）。

## 3. 功能需求叢集（Feature Request Clusters）

去重歸類後依頻率排序。

### 3.1 學習統計／進度儀表板（含 streak）— frequency 5
- **affectedPersonaIds**：p1, p2, p3, p4, p5（全員）
- **JTBD**：用戶要「感覺到自己在累積／進步」，以說服自己續訂、也作為向他人推薦的背書素材；月底扣款時需要一個「理所當然按續訂」的理由。
- **代表性引言**：
  - p5：「展示『本週聽了 120 分鐘、新增 12 個單字、連聽 5 天 streak』，讓我能『看到』進度。」
  - p4：「顯示本週聽幾集、連續打開幾天，像 Duolingo，但不要那麼幼稚／複雜。」
- **buildComplexity**：medium（埋點 + 數據聚合 + 新頁面；streak 需定義「今天算學了」門檻防作弊）

### 3.2 內容主題／情境分類篩選 — frequency 4
- **affectedPersonaIds**：p1, p2, p3, p5
- **JTBD**：快速找到與自身工作／目標相關的集數（職場、商務、面試、時事），不必從標題猜、不必滑時間軸。
- **代表性引言**：
  - p1：「我想要可以直接篩選『職場英語』的集數。」
  - p5：「商業、日常、時事，讓我找到跟我工作最相關的。」
- **buildComplexity**：medium（內容打標籤 + 篩選 UI；曲庫頁同步支援）

### 3.3 閃卡改造：例句脈絡 + 彈性排程 + 淺層入口 — frequency 4
- **affectedPersonaIds**：p1, p3, p4, p5
- **JTBD**：讓複習有記憶鉤子（背面顯示原始例句與來源集數）、不形成欠債壓力（可設每日上限／一鍵延後）、且每天能低摩擦進入（入口上移到首頁／底部導航）。
- **代表性引言**：
  - p1：「翻到背面要直接顯示那個單字在哪集的原始句子，不然背完就忘。」
  - p5：「讓我設每天最多複習 N 張，或一鍵把今天的卡全部推後一天。」
- **buildComplexity**：medium（例句綁定＝資料結構調整；排程彈性＝SRS 演算法調整；入口＝導航改版）

### 3.4 車用／純音頻模式 + 自動續播 — frequency 2
- **affectedPersonaIds**：p2, p5（開車族）
- **JTBD**：零操作、不看螢幕也能持續聽與學，讓 app 成為通勤背景音而非需手動管理的工具。
- **代表性引言**：
  - p2：「進入後螢幕可以關掉或只顯示大字幕，不需要手動操作……播完一集自動播下一集。」
- **buildComplexity**：medium（純音頻 UI + 自動續播佇列；TTS 重點單字複習屬 high）

### 3.5 句子／整句收藏與快速標記（bookmark）— frequency 2
- **affectedPersonaIds**：p1, p5
- **JTBD**：聽到好句但當下不便查詞（開車／晃動），想快速標記稍後回看，或長按字幕存整句（不只存單字）。
- **代表性引言**：
  - p1：「我想直接長按字幕存整句，不是只能存單字。」
  - p5：「我想要一個『快速標記』讓我回來再看，現在只能整集重聽。」
- **buildComplexity**：low（沿用單字本資料結構擴充句子型別 + 長按手勢）

### 3.6 每集後測驗題（選擇／填空）— frequency 1（具代表性，學生族）
- **affectedPersonaIds**：p3
- **JTBD**：備考族需要「聽完馬上測」的即時反饋與備考感，否則覺得只是「聽過了」、CP 值低。
- **代表性引言**：
  - p3：「每集後面加 3～5 題……聽完馬上測驗，這樣我才會覺得有在備考。」
- **buildComplexity**：high（逐集生成題目 + 作答 UI + 結果回饋）

### 3.7 每日一集推薦（Today's Pick）— frequency 1（具代表性，選擇困難族）
- **affectedPersonaIds**：p4
- **JTBD**：消除「打開 app 要自己滑、選不定就關掉」的瀏覽摩擦，給選擇困難用戶一個直接進入點。
- **代表性引言**：
  - p4：「打開 app 直接顯示『今日推薦，點播』，省掉瀏覽選擇的摩擦。」
- **buildComplexity**：medium（推薦邏輯 + 首頁固定模組）

### 3.8 AI 語音自然度提升 — frequency 1（具代表性，影響信任）
- **affectedPersonaIds**：p2
- **JTBD**：語調太像唸稿會侵蝕「這是真實英語」的信任感，影響長期黏著。
- **代表性引言**：
  - p2：「AI 語音有沒有辦法停頓一下、讓語調更像人在說話而不是在唸稿？這個影響我的信任感很大。」
- **buildComplexity**：high（TTS 模型／後製，非前端範圍）

### 3.9 播放速度跨集記憶 — frequency 2（同時是 bug 修復）
- **affectedPersonaIds**：p3, p5
- **JTBD**：不想每集重設速度，希望系統記住偏好。
- **buildComplexity**：low（設定持久化修正）

## 4. 流失風險訊號（Churn Risk Signals）

### 4.1 留訂理由是消極的（沉沒成本／懶得取消／沒更好替代品）
- **頻率**：5/5
- **affectedPersonaIds**：p1, p2, p3, p4, p5
- **量化佐證**：renewalLikelihood 平均 ≈ 6.2，但語言層面普遍脆弱
- **訊號**：
  - p1：「我留著不是因為很喜歡，是因為還沒找到更好的而且 149 在我接受範圍內。這是很脆弱的理由。」
  - p5：「現在的狀態是沉沒成本在撐著，不是真心覺得值。」
- **mitigationHint**：用進度儀表板把留訂理由從消極（沒替代品）轉為積極（看得到投入與進步）；月底前主動呈現「本月你聽了 X 分鐘、學了 Y 字」作為續訂正當性。

### 4.2 SRS 閃卡全員停用 = 失去每日回訪行動
- **頻率**：5/5（全員 srsStatus 都已停用，多在第 3 週前後）
- **affectedPersonaIds**：p1, p2, p3, p4, p5
- **量化佐證**：flashcardEffectiveness 平均 ≈ 4（p1=2 最低）
- **訊號**：閃卡是唯一「不依賴新內容、每天可做」的行動，全員放棄＝週更 2 集之外無回訪鉤子。
- **mitigationHint**：閃卡背面加例句脈絡（提升回報）、加休息日／每日上限（移除欠債壓力）、入口上移到首頁或底部導航（降低摩擦）；把「複習今集單字」設為播放結束後的明確下一步。

### 4.3 進度感不足 → 不知道是否進步 → 不敢推薦
- **頻率**：5/5（progressPerceptionGap 全員皆答「不確定有變好」）
- **affectedPersonaIds**：p1, p2, p3, p4, p5
- **量化佐證**：wouldRecommend 僅 p4=true，其餘 4 人皆 false；npsProxy 平均 ≈ 6
- **訊號**：
  - p3：「沒辦法跟朋友解釋付費理由。」
  - p1：「在我能確認自己進步之前，我不敢推薦。」
- **mitigationHint**：進度可視化同時解決留存與口碑；提供可分享的成就截圖（本週 streak／學習時數），製造病毒式傳播素材。

### 4.4 帳單扣款 × 低使用週 = 取消意圖觸發點
- **頻率**：4/5（p1、p3、p4、p5 都在「兩週沒開後看到扣款」當下動念取消）
- **affectedPersonaIds**：p1, p3, p4, p5
- **訊號**：
  - p1：「看到帳單扣款 149，第一個反應是『我是不是在白繳？』」
  - p5：「算了一下單集成本比 Spotify 貴，認真想過不續。」
- **mitigationHint**：低使用週後主動 re-engagement（推播「這週還沒開，週六新集等你」）；扣款前夕呈現累積價值摘要，扭轉「白繳」感知。

### 4.5 習慣完全綁定外部情境，情境一斷即流失
- **頻率**：5/5（出差、忙碌專案、婆婆來住、考前衝刺都導致連續多天不開）
- **affectedPersonaIds**：p1, p2, p3, p4, p5
- **訊號**：
  - p5：「脫離通勤情境就會斷掉……app 本身沒有任何提醒或 re-engagement 機制。」
