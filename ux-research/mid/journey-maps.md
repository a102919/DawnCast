# DawnCast 付費用戶旅程地圖（中期評估性研究）

> 研究階段：中期評估性研究（付費訂閱 3 個月後）
> 訪談日期：2026-06-20
> 對象：4 位 validated persona（P1 林佳穎 / P2 陳冠廷 / P3 黃詩涵 / P4 蘇怡君），皆為現役付費訂閱者
> 框架：6 階段旅程（awareness / consideration / first_use / habit_formation / renewal_decision / churn_risk），emotionScore 1–5

---

## P1 林佳穎 — 行銷專員，捷運通勤 25 分鐘，B1 職場英語

**整體弧線（overallArc）**：Zoom 糗況觸發學習焦慮 → 試用時查詞 popup 輕快體驗加上 149 元低門檻情緒衝動升級 → 初期職場集數閉環產生真實成就感 → 閃卡第三週失去脈絡全員放棄，習慣回訪鏈斷裂 → 兩週低頻期撞上帳單扣款，靠「下週有跨國電話要開」功能性理由留存 → 續訂意志脆弱，一旦出現替代品或連續幾週沒感興趣主題即流失

**關鍵時刻（criticalMoment）**：第二個月底連續兩週未開 app，收到 NT$149 扣款簡訊，當下自問「我是不是在白繳？」認真想過取消；最後沒取消是因為下週有跨國電話會議，想讓耳朵熱身——是功能性理由支撐而非積極喜愛。這個「被動留存」時刻是最決定性的流失預警：app 沒有任何量化數據可以在用戶自問「有沒有在用」時提供答案，一旦短期內沒有外部會議壓力，流失隨時發生。

### Stage 1：Awareness（emotionScore：3）

| 面向 | 內容 |
|------|------|
| **使用者行動** | Zoom 英語會議腦袋空白三秒後，當天在捷運站滑手機偶然發現 DawnCast；下載 Free 版，在捷運上聽了半集「職場英語」主題，確認速度跟得上 |
| **thoughts** | 「AI 雙人對話感覺比一般唸稿有生命，25 分鐘通勤剛好。但首頁一堆鎖頭，不確定裡面有沒有我需要的開會、email 職場細分主題。」 |
| **emotionLabel** | 謹慎期待 |
| **painPoints** | 首頁集數卡：鎖頭集數視覺佔比大，但主題標籤僅有 CEFR 等級，無法判斷曲庫是否持續有「開會英文」「email 用語」等職場細分主題 |
| **opportunities** | 首頁集數卡：鎖頭集數保留鎖定狀態，但顯示第一層情境 tag（開會 / 職場 email / 求職 / 出差），讓試用戶在付費前評估曲庫相關性——tag 資料已存在，屬呈現層調整 |

### Stage 2：Consideration（emotionScore：4）

| 面向 | 內容 |
|------|------|
| **使用者行動** | 試聽到「如何在 email 婉拒邀請」集數中段被鎖住；查了一個詞的 popup 跳出來乾淨俐落；看到月費 149，情緒驅動當場升級 |
| **thoughts** | 「那集剛好是我當下最需要的情境，氣不過加上 149 不貴，直接按下去了。查詞 popup 那個體驗是最後臨門一腳，不是功能清單感動我，是那個體驗在對的時機出現。」 |
| **emotionLabel** | 挫折轉行動，情緒衝動升級 |
| **painPoints** | 方案頁：升級 CTA 下方沒有「升級後繼續播放你剛才在聽的那集」的明確說明，情緒高峰過後升級理由模糊；Free 查詞 3 次限制觸發了升級，但用戶不清楚付費後能解鎖哪些職場情境集數 |
| **opportunities** | 升級完成後導回頁：直接跳回中斷的播放器頁並從中斷點續播，讓用戶的第一個 Pro 動作就是「完成剛才被打斷的體驗」；方案頁 CTA 改為「解鎖這集，繼續收聽」而非通用文案 |

### Stage 3：First Use（emotionScore：4）

| 面向 | 內容 |
|------|------|
| **使用者行動** | 升級後在捷運上聽完那集；查詞 popup 每集使用多次；有一集「如何婉拒邀請」，查了 take that on 加入單字本，當天下午在工作 email 真的用到，記住了 |
| **thoughts** | 「無限查詞是真的解放，那集講 email 婉拒，查了 take that on，當天下午 email 就用了，真的記住了。查完馬上跳出來不延遲，這個體驗比想像中好很多。」 |
| **emotionLabel** | 投入，「值了」瞬間 |
| **painPoints** | 播放器頁字幕區：捷運晃動時容易誤觸字幕上的單字，popup 跳出後要關掉需點叉叉，干擾播放節奏；誤觸率在非手持情境（手機斜放口袋露出）下更高 |
| **opportunities** | 播放器頁：popup 加入 2 秒無操作自動收起（可設定），減少誤觸的後續操作成本；字幕區誤觸 popup 時保持播放繼續，不自動暫停（目前是否暫停邏輯不確定，需驗證） |

### Stage 4：Habit Formation（emotionScore：2）

| 面向 | 內容 |
|------|------|
| **使用者行動** | 前兩週每天通勤聽，查詞存單字本；第三週打開閃卡複習頁，看到 20 幾張來自不同集數的單字混雜（quarterly review、leverage、synergy），無法辨識對話脈絡，關掉；此後閃卡再也沒開，習慣回訪鏈斷裂 |
| **thoughts** | 「閃卡跳出 20 幾張，全混在一起，不知道哪集哪個情境，背起來很空洞。點『不認識』，幾天後再出現時已忘記為何收藏。通勤時間到了我還在刷卡，就覺得我是來練聽力的不是來背單字的，之後就不管閃卡了。」 |
| **emotionLabel** | 習慣初建，複習系統失效 |
| **painPoints** | 閃卡複習頁：（1）每張卡片無來源集數與對話脈絡資訊，單字脫離情境背不起來；（2）20 幾張一次呈現帶來量感壓迫；（3）閃卡入口藏在「單字本 → 點按鈕」二級頁面，底部導航無直接入口，找不到按鈕在哪 |
| **opportunities** | 閃卡複習頁：（1）卡片翻面顯示「來自第 N 集：[集數標題]」加原始例句（字幕資料已存在，需關聯）；（2）每日預設上限 10 張，超出顯示「今日已完成，明天繼續」；（3）首頁顯示「今日待複習 N 張」入口卡片，讓複習成為開 app 的自然次要動作 |

### Stage 5：Renewal Decision（emotionScore：2）

| 面向 | 內容 |
|------|------|
| **使用者行動** | 連續兩週未開 app；收到帳單扣款 149，第一反應「我是不是在白繳」；進設定頁找取消按鈕，找了「訂閱管理」進去沒有明確取消流程；最後因為下週有跨國電話會議決定留下 |
| **thoughts** | 「149 不是問題，問題是有沒有在用。app 完全沒有告訴我我這個月聽了幾集、學了幾個字，什麼數字都沒有，我只能靠感覺，感覺說我懶得取消就算了。留下來是功能性的理由，不是因為很愛這個 app。」 |
| **emotionLabel** | 猶豫，被動留存 |
| **painPoints** | （1）整個 app 無學習統計數據，用戶在自問「有沒有在用」時得不到任何量化答案，只能靠主觀感覺；（2）設定頁「訂閱管理」點進去無明確取消操作，用戶意圖無法完成（dark pattern 殘影，用戶感受到不被信任） |
| **opportunities** | （1）月底扣款前 3 天推送「本月學習摘要」（聽了 X 集 / X 分鐘 / 查了 X 個詞 / 新增 X 個單字），用現有播放與查詞資料幫用戶回答「有沒有在用」；（2）設定頁「訂閱管理」清楚顯示目前方案、下次扣款日、取消按鈕，讓用戶可以完成取消意圖——信任感的建立反而能降低流失焦慮 |

### Stage 6：Churn Risk（emotionScore：2）

| 面向 | 內容 |
|------|------|
| **使用者行動** | 第三個月繼續訂閱，但明確設了取消條件：連續幾週沒有感興趣主題、或一週開不到 2 次；首頁看到不感興趣的集數直接關掉，已多次發生 |
| **thoughts** | 「下個月應該還會續，但如果連續一個月我開不到 2 次，我會取消。或者如果出現更好的替代品。我不是因為喜歡 DawnCast 留著，是因為還沒找到更好的而且 149 在接受範圍內。這是很脆弱的理由。」 |
| **emotionLabel** | 條件式留存，高流失風險 |
| **painPoints** | 首頁集數列表：無主題情境篩選，想找「開會用英文」相關集數要一集一集翻；主題不感興趣時沒有「看看其他相關集數」的引導，開啟即放棄，直接關 app |
| **opportunities** | 首頁集數列表：在 CEFR 等級篩選旁加入情境 tag 橫向捲動篩選列（開會 / 職場 email / 出差 / 電話英語 / 求職），tap tag 即篩選；這是現有集數標籤資料的 UI 呈現層改善，不需要新資料結構 |

## P2 陳冠廷 — 竹科硬體工程師，開車通勤 40 分鐘，B2 科技商業英語

**整體弧線（overallArc）**：科技主題集數鎖頭觸發升級 → 查詞 popup 在停車場精聽建立核心正向飛輪 → 閃卡因積壓卡片壓力第 10 天中斷 → 第六週 AI 語音朗讀感觸發信任危機，差點流失 → 靠科技主題命中救回，條件式續訂 → 留存完全依賴內容策略，車用使用情境的設計缺口持續存在

**關鍵時刻（criticalMoment）**：第六週連續出現軟性生活主題，幾乎沒打開 app；同週 AI 語音朗讀感觸發「這跟直接聽 BBC 有什麼差」的核心質疑——差點取消。最後靠第三週一集 semiconductor 相關集數救回。這個時刻揭示留存完全外掛在內容策略，app 本身對低主題興趣週無任何主動挽留設計；任何連續兩週非科技主題就能把陳冠廷推出去。

## P3 黃詩涵 — 大學生，備考多益 B1→B2，公車通勤 20 分鐘

**整體弧線（overallArc）**：備考壓力下 FOMO 衝動升級 → 查詞功能完全符合期待，前兩週高度投入 → 閃卡無提醒機制自然消亡，單字本淪為「只進不出的垃圾桶」 → 主題不符加上單字堆積觸發罪惡感，第六週跌至流失邊緣 → 靠一集考試技巧主題拉回，但 app 無任何考試導向功能，備考用途定位模糊

**關鍵時刻（criticalMoment）**：訂閱第六週多益考前兩週，打開 app 滑了一下，發現全都是聽力輸入，沒有練習題，沒有測驗——自問「這個幫我考試的 CP 值好低」，當下差點去設定頁取消。最後沒取消是因為 149 不貴且懶得走取消流程，同時怕砍掉後想聽某集又要重新付。留存靠的是取消摩擦力而非正向體驗，這是最決定性的流失預警：任何讓取消摩擦力降低的動作（如清楚的取消按鈕或簡化流程）都會加速流失。

## P4 蘇怡君 — 全職媽媽（前護理師），碎片化家務背景聆聽

**整體弧線（overallArc）**：低壓期待加上鎖頭 FOMO 衝動升級 → 雙語字幕降低焦慮、查詞流程超預期流暢 → 閃卡積壓壓力第三週觸發一次性關閉，此後再未開啟 → 孩子生病兩週斷線、靠偶然內容命中拉回 → 三個月後自己都驚訝還在續訂，但完全無法量化進步，留存純靠輕壓力定位和取消摩擦

**關鍵時刻（criticalMoment）**：婆婆來住、孩子生病期間，連續兩週無法開 app，重新打開時有「我果然沒恆心」的低落感；最後因新一集封面主題有趣、聽了覺得「欸還好我還是喜歡這個」才沒去取消。這個偶然的內容命中是唯一留存觸發點——app 本身在中斷期後沒有任何主動挽留設計，每次重開都是一場賭局：封面主題夠不夠有趣。
